Rules

Rules refer to the options you can pass to whitelister instructing it how to filter and validate input. Some rules apply to all types while other rules are specific to one or more types.

Rules are usually an object, for example: {% highlight js %} const rules = { user_id: { type: 'integer' }, email: { type: 'email', required: true }, }; {% endhighlight %} If you only need to pass a type rule, use shorthand: {% highlight js %} const rules = { user_id: 'integer', email: { type: 'email', required: true }, }; {% endhighlight %} Finally, you can pass a "custom" rule in the form of a function: {% highlight js %} const rules = { user_id: { type: 'integer' }, email: (val) => { if (val === 'david@spire.me') { return 'david@spirelabs.co'; } return val; }, }; {% endhighlight %}
